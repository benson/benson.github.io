# Runtime enemy asset generation

These are separate gameplay cutouts, not crops of the square field-guide art.
They were generated with the built-in image-generation tool, keyed on a flat
green background, locally converted to alpha, tightly cropped, and exported as
alpha WebP. Each corresponding `assets/guide/enemies/*.webp` image was supplied
as a visual reference only.

## Exact prompts

### Skitter

```text
Use case: stylized-concept
Asset type: browser action-game runtime enemy cutout sprite
Input image: visual reference for creature identity, silhouette, materials, and red/black palette only
Primary request: Create a production-ready full-body runtime sprite of this small four-legged mechanical skitter drone. Recompose it into a clean 3/4 top-down isometric game view, facing toward screen-right. Make the silhouette instantly readable at 48–64 pixels: pointed armored head, one bright red optic, four long blade-like legs, compact low body. Preserve the Lastlight visual language: rugged near-future salvage machinery, angular plated armor, graphite joints, red hazard glow, hand-painted high-end tactical game concept art. Strong material separation and restrained edge highlights.
Composition/framing: entire creature visible, centered, generous even padding, no cropping; no ground or environment.
Scene/backdrop: perfectly flat solid #00ff00 chroma-key background for removal. The background must be one uniform color with no shadows, gradients, texture, reflections, floor plane, or lighting variation.
Lighting/mood: directional cool rim light plus subtle warm red internal glow; legible, ominous.
Constraints: single creature only; crisp opaque outer edges; no cast shadow; no contact shadow; no reflection; no particles; no smoke; no text; no logo; no watermark; do not use #00ff00 anywhere in the subject.
```
