  1. On GitHub, open Settings → Password and authentication.

05

Every change

Change, check, and ship

Pull→Ask Claude→Review→Check→Push→Verify

1. Start clean and pull Benson’s latest work

Set-Location "$env:USERPROFILE\Projects\benson.github.io"
git switch master
git status --short
git pull --ff-only origin master
claude

If git status --short prints unfamiliar files, stop. Ask Claude to explain them. Don’t delete, reset, or overwrite them.

2. Give Claude the mission

Replace the bracketed line with the actual change.

I am Jacob, working under Benson's standing authorization to make this
Lastlight change directly on master. Work only inside lastlight/**.

Task: [DESCRIBE THE CHANGE AND WHAT “DONE” LOOKS LIKE]

Before editing, confirm the branch is master, the tree is clean, and
origin/master is pulled with --ff-only. Implement and test the change, run
the complete Lastlight GitHub Actions check suite, and review the final diff.

Never force-push, discard work, or touch other project folders. Stop on a
conflict, a check you cannot fix, or an unexpected file outside lastlight/**.

When everything is green, make one focused commit, synchronize with the newest
origin/master, push directly to master, watch Lastlight Checks and GitHub Pages,
and verify the live game. If lastlight/worker/** changed, test and deploy the
Cloudflare relay separately, report its version, and verify /health.

3. The checks that guard the live game

Claude should run all of these before pushing:

npm --prefix lastlight run check
npm --prefix lastlight run sprites:verify
npm --prefix lastlight run motion-atlases:verify
npm --prefix lastlight run motion-atlases:test
npm --prefix lastlight test
npm --prefix lastlight run fixtures:verify -- --report=artifacts/fixture-report.json
npm --prefix lastlight run soak:ci -- --report artifacts/multiplayer-soak.json
npm ci --prefix lastlight/worker
npm test --prefix lastlight/worker

Why local checks matter: GitHub Pages starts publishing as soon as the push lands. The online checks do not currently hold the deployment back.

4. Review, commit, synchronize, push

git status --short
git diff
git add -- lastlight
git diff --cached
git commit -m "Lastlight: describe the change"
git pull --rebase origin master
git push origin master

Do not use git add . in this multi-project repository. Stage only Lastlight.

07

If something goes wrong

Recover safely

My push was rejected

Benson probably pushed first. This is normal. Never add --force.

git pull --rebase origin master

Resolve with Claude, rerun every check, then push normally.

A bad website commit is live

Make a new commit that reverses it. This keeps history intact.

git log --oneline -10
git revert BAD_COMMIT_ID
git push origin master
A bad relay deployment is live

Tell Benson immediately, then roll back and verify health.

Set-Location "$env:USERPROFILE\Projects\benson.github.io\lastlight\worker"
npx wrangler deployments list
npx wrangler rollback
Claude wants to touch another project

Stop it. This collaboration is scoped to lastlight/**. Other folders publish unrelated parts of Benson’s website.

The two never-commands: never force-push shared master, and never use git reset --hard to solve a collaboration problem.
